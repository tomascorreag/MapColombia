# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Digital Humanities project: an interactive browser-based map of Colombia correlating two georeferenced, timestamped datasets:

1. **Violence events** — masacres and other violent events of the armed conflict.
2. **Election results** — Chamber, Senate, and Presidential results.

The artifact is a web page that displays both layers dynamically (map + time dimension) so spatial/temporal correlation between violence and electoral outcomes can be explored.

A third, **experimental deforestation view** ships alongside, gated behind a URL flag (`?section=deforestation`) — a year-scrubbed pixel raster of forest-cover loss. It reuses the same map shell and time-scrubber but is otherwise self-contained (see **Deforestation view** below).

## Architecture

- **`pipeline/`** — Python (pandas/geopandas) for acquisition, cleaning, geocoding, and joining. Outputs clean GeoJSON/CSV consumed by the frontend. Raw downloads go in `data/raw/` (never hand-edited); processed outputs in `data/processed/`.
- **`frontend/`** — Svelte 5 (runes) + TypeScript + Vite; MapLibre GL JS v5 + deck.gl v9 via `MapboxOverlay` (direct, no wrapper lib); CARTO dark-matter basemap for dev (production plan: Protomaps PMTiles); D3 charts in Svelte components. Full rationale and **load-bearing constraints** (float32 time encoding, filterRange-only scrubbing, feature-state choropleths, no terrain) in `docs/stack-decision.md` — read it before touching map code.
- **Hosting** — app shell on GitHub Pages; `.pmtiles` + large data on Cloudflare R2 (GitHub Pages range-request bug with PMTiles, protomaps/PMTiles#584).

## Build & run

- Pipeline: `python pipeline/build_frontend_data.py` → `data/processed/frontend/` (violence.bin binary columnar + violence_details.bin gi-aligned per-event victim portrait, lazy-loaded by the click panel + elections.json + munis.json + munis_shapes.json + memoria.json). Requires raw data (see Data sources) and `geopandas pyogrio shapely>=2.1` for the polygon step.
- Frontend: `cd frontend && npm run dev` (predev hook copies `data/processed/frontend/` → `public/data/`; that copy is gitignored — the canonical artifact is in `data/processed/`).
- Checks: `npm run check` (svelte-check + tsc), `npm run build`; visual smoke tests `node scripts/smoke.mjs` / `smoke-deforestation.mjs` / `smoke-landing.mjs` against a dev server on :5199 (Playwright); perf probe `node scripts/probe-fps.mjs` (headed — headless WebGL is software-rendered). `smoke-landing.mjs` gates the landing's two ambient canvases on sampled pixels (a blank field throws nothing), asserts the landing fetches **zero** archive data, and asserts the OS reduce-motion pref is **ignored** (owner decision: the `app.css` clamp is commented out and the JS guards in `Landing.svelte`/`App.svelte` are pinned `false` — the gate checks `animation-delay` is *not* clamped, so re-honoring the pref means flipping all four sites together). Note `vite preview` needs `--base /MapColombia/` (the config only sets `base` when `command === 'build'`, so preview otherwise 404s the bundle).
- The Memoria tab's left–right scores come from the curated, cited table `pipeline/party_lr.py` — scholarly interpretation, flagged for owner review; the per-row citation is the contract. Never add a score without one.
- Deforestation pipeline (separate from the main build): download steps (all no-auth) `python pipeline/download_hansen.py` (Hansen GFC-2025-v1.13 `lossyear` tiles → `data/raw/hansen/`), `download_treecover.py` (Hansen `treecover2000` canopy-% tiles, same no-auth bucket/tiles — feeds the jungle-green forest backdrop), `download_drivers.py` (WRI driver raster), and the Phase 2 attribution sources `download_corine.py` (IDEAM CORINE 2022 ag/pasture polygons, REST GeoJSON), `download_coca.py` (UNODC SIMCI coca density), `download_ica_bovino.py` (ICA cattle census `.aspx` Excel), `download_boundaries.py` (RUNAP + Ley 2ª legality layers, REST GeoJSON). Then `python pipeline/build_deforestation.py` → `data/processed/frontend/deforestation_lossyear.png` (legacy ~695 m coarse fallback, R=year/G=density/B=packed codes) + `deforestation_forest.png` (R=year-2000 canopy %, A=country mask — backdrop) + `deforestation.json` (per-municipio/national stats + curated meta). Requires `rasterio numpy pillow xlrd openpyxl` and the already-built `munis.json`/`munis_shapes.json`. **Then `python pipeline/build_deforestation_tiles.py`** → `deforestation_lossyear.pmtiles`, the near-native Web-Mercator pyramid the frontend actually renders (z5..z12, ~38 m finest; reuses `build_deforestation.py`'s loaders/constants; adds the `pmtiles` pip dep). **Tiles are lossless WebP** (not PNG) — bit-exact so the categorical R=year/B=packed channels are preserved, but ~2x smaller than PNG on this sparse raster (~89 MB vs ~172 MB); the frontend decodes via `createImageBitmap`, which sniffs the format, so no client change is tied to the encoding. The `.pmtiles` is **committed** (~89 MB) — it must be in git, because CI builds from a plain checkout and `prebuild`'s `sync-data` can only copy what is on disk; gitignoring it silently shipped a `dist` without it (404 → loss raster drew nothing while the committed forest PNG and JSON stats still worked, so the map looked half-alive). Rebuild via the script and re-commit; `sync-data` copies it into `public/data` for dev. Range-requested, so only viewport tiles are fetched (lazy) — visitors never pull the full 89 MB from a host that honors Range. Served from Pages today despite the PMTiles-on-Pages range bug; **`VITE_TILES_BASE` moves just this file to R2** (or any range-reliable, CORS-enabled origin) as a build-time config change if the bug bites. Tradeoffs and rejected options: `docs/stack-decision.md` §3. Curated headline figures + per-row citations (IDEAM totals, WRI drivers, CORINE ag-kinds, SIMCI coca, ICA cattle, RUNAP/Ley2 legality) live in `pipeline/deforestation_meta_curated.py` — owner-review-flagged, same contract as `party_lr.py`. Phase 2 spec/realized notes: `frontend/DEFORESTATION_PHASE2_PLAN.md`.

## Landing page

`frontend/src/lib/Landing.svelte` at bare `/` (no `?section=`). Self-contained: no map, no deck.gl, and **no data fetch** — `App.svelte` short-circuits the load when `section === null`, and `smoke-landing.mjs` gates that it stays that way.

- **Palette is quoted, not invented.** One shared `--ink` ground; no hue split. An earlier build divided the page red-vs-green, but neither hue came from the archives (the deforestation view's green canopy exists to be *burned*, and its `#5fb878` appeared in no shader) — and red/green is the deuteranopia confusion axis, a poor basis for the one decision the page exists to support. The halves are told apart by the **form of their mark**: crimson strands vs blocky amber pixels. Every colour is an existing token or a verbatim shader quote (`#601016`/`#ff3a1c` from `TendrilExtension.ts`; the `#ab8c87`→`#d11c0d`→`#ff8a1f`→`#fff1c2` ramp from `fireRamp.ts`). `--gold` belongs to neither side, so it carries the shared chrome. Colour stays **out of the type** — only the CTA takes an accent, on hover.
- **`landingEffects.ts` is atmosphere, not cartography** — read its header before touching it. Both fields are canvas-2D ports (`createScarField` / `createEmberField`) whose wounds, loss years and densities are **synthetic**, from a seeded PRNG and value noise. They are masked to each half's **outer flank** and must never overlap the Colombia silhouette: strands or embers laid across the country outline would read as real violence/deforestation mapped onto Colombia, which is exactly the claim they cannot make. **On stacked layouts (`max-width: 760px`) the fields are not rendered at all** — the silhouette then spans the middle and no placement clears it, so the atmosphere goes and the thesis stays.
- Shared maths lives in `noise.ts` (`mulberry32`/`valueNoise`/`flowAngle`, extracted verbatim from `tendrils.ts`) and `fireRamp.ts` (`RAMP_DEFAULTS`/`FIRE_DEFAULTS`, split out of `LossRasterLayer.ts`, which re-exports them so no call site changed). `fireRamp.ts` exists so the landing can quote the ramp **without importing deck.gl**. Constants in `landingEffects.ts` are marked `[verbatim]` (quoted from the real layer) or `[landing]` (retuned for a looping 2D field — mostly timing, plus three documented deviations: cold embers fade where the real layer must keep them for cumulative extent; scar alpha is raised because a few hundred curves never accumulate like the real 80k; and the ember field samples the ramp only across `RAMP_LO..RAMP_HI` (0.3..0.92), since the full range's ends are clay `#ab8c87` — an established-land claim the landing doesn't make — and cream `#fff1c2`, bright but colourless, both of which wash the field grey. That window invents no colour: the ramp is a continuous interpolation of the quoted stops, so it only picks *where on the quoted ramp* to sit).
- Motion: a slow dim loop at rest, igniting on **cursor proximity** (not hover). `ignite` is a continuous 0..1 falloff from each field's **mask-centre anchor** — `FIELD_ANCHOR` in `Landing.svelte` must track the `mask-image` ellipses in `.violence .fx`/`.ashes .fx`, since the canvas fills its half but only the outer-flank ellipse is revealed (anchoring on the canvas box would peak the glow over blank ink). Tracked on `<svelte:window>`, because the masthead/silhouette/footer overlay `.split` and would eat its pointermoves; anchors are cached in `fit()` so the hot path forces no layout. Keyboard focus pins ignite to 1 (no coordinates to fall off). Ignite drives speed (~13x rest→hot), glow, stroke width, the type's scale, and **reach** together; **only rates and [landing] look knobs move** — the verbatim `COOL`, taper and `FADE_DAYS/WOUND_PERIOD_DAYS` ratios are untouched.
- **Reach** (`FIELD_REACH_*`, `maskRadius`) is the field growing across its flank: ignite pushes it 0.4→0.82 of the mask ellipse, and the **CSS mask is the outer bound** (it fades out by 0.74). `MASK_*` in `landingEffects.ts` mirrors the `mask-image` ellipses in `<style>` — one geometry in two places, change one change both. **That ellipse is the type's budget, and it is nearly spent.** It was 56%x54%, tuned until its rim just grazed the title; it is now 70%x62% so the fields reach much further on hover, which puts the gradient's faint outer rim ~100px under the title's left edge and ~54px into the content block — verified legible, but there is little room left. Push it much past this and the type goes. Scars apply reach as **one `destination-in` pass** over the composited frame, not a per-segment gate: the scars are a pre-rasterised blit, so a per-segment test could only reach the flares and would leave the veins at full extent giving the size away at rest. Embers apply it per cell, since that loop already writes alpha. Scars *also* gate the flare on `segArc` so light runs out along each strand. Neither field rebuilds geometry.
- Ember noise splits **px vs cells, and the split is load-bearing**: `COV_LEN_PX`/`YEAR_LEN_PX` are *composition* (shape size) and live in CSS px, so refining `CELL_PX` no longer shrinks every patch with it; `BURN_LEN_CELLS`/`GRIT_LEN_CELLS` are *texture* and must stay in cells. Pinning `BURN_LEN` to px is a real bug, not a preference — at a 2px cell a 24px wavelength is 12 cells, which ignites whole neighbourhoods in unison and melts the ragged front into smooth blobs.
- Type tracks ignite too: the rAF loop writes the same eased value to a `--ignite` CSS var on each `.half`, and `.content` scales `1 + 0.035*ignite` from its outer bottom corner. Not a `:hover` transition — that runs on its own clock, drifts out of step with the canvas, and snaps binary. The write is quantised (`toFixed(3)`) and diffed against the last value, because `igniteNow` approaches its target asymptotically and would otherwise invalidate style every frame forever on an idle page. The flare **bloom is stroked in `SCAR_RGB`, never the flare colour**: `lighter` sums channels, so blooming with `#ff3a1c` saturates R and drags G/B up until the wound goes cream-white and the violence half reads as *fire* — the other archive's language. Scars rasterise **once** to an offscreen canvas (they're permanent — that's the semantics, not an optimisation); only flaring wounds stroke per frame. **`WOUND_PERIOD_DAYS` (1500) is how often the field lights up and is the frame budget's tightest knob** — `FADE_DAYS/PERIOD` is the concurrency: halving it to fire more often doubled the stroke count and broke the frame outright (33 ms ignited, 30 fps), and is affordable only because the bloom gained a visibility cull (`HALO_MIN_ALPHA`) that restored 16.7 ms with no visible change (peak flare energy 970 vs 972). It strokes at 3.4x width, so culling it where it can't be seen is nearly free frames. Look here first if the landing ever drops frames. Both fields loop by per-element modulo, never a global playhead reset. The ember mark stays **blocky** — `CELL_PX` is 2, a finer pixel and never a smooth one, since "blocky amber pixels vs crimson strands" is what tells the halves apart and the real layer mandates nearest filtering; 2px is arguably nearer the real view, whose finest tiles are ~38 m/px. That took the cell count to ~169k, paid for by rebuilding the per-cell `pow`/`smoothstep` curves as 256-entry **per-frame LUTs** (`refreshLuts`) — heat and coverage quantise to 8 bits on output anyway, so the table costs no fidelity. Frame interval is unchanged rest vs fully ignited (60 fps in both). Deliberately does **not** import `perf.svelte.ts` — its tier sniff spins up a throwaway WebGL context at module load.

## Deforestation view

Experimental third view, additive to the violence/elections code (a third `app.tab` value, `'deforestation'`). Activated only by `?section=deforestation` in the URL — read once at startup in `App.svelte` (no router; works on GH Pages project sites where a path would 404). Plain `/` is unchanged.

- **Render** — Hansen tree-cover-loss as a GPU-thresholded **pixel raster**, not a choropleth. `frontend/src/lib/LossRasterLayer.ts` is a custom `BitmapLayer` subclass; a deck.gl `TileLayer` (`@deck.gl/geo-layers`) instantiates **one per tile** over a Web-Mercator **PMTiles pyramid** `deforestation_lossyear.pmtiles` (z5..z12, finest ~38 m/px ≈ native 30 m; built by `pipeline/build_deforestation_tiles.py`, read client-side via the `pmtiles` npm lib in `TileLayer.getTileData`). This replaced the old single full-country texture, which could not reach 30 m (one GL texture caps ~115–230 m). Each tile texture is: R=earliest loss year, G=loss density, **B=PACKED codes** (bits 0-2 WRI driver, 3-4 ag-kind, 5-6 legality, 7 coca-present), A=mask. All four attribution codes are bit-packed into B because a **second BitmapLayer sampler fails luma's `_areTexturesRenderable` and silently skips the whole draw** (empty map, no console error) — packing keeps **each tile a single-texture draw** (the luma limit is per-layer, so per-tile is fine); **nearest filtering is mandatory** so the packed byte isn't interpolated. A std140 uniform (`maxYear = defPos - 2000`) discards pixels above the scrubbed year and G-density drives opacity (honesty fix — a naive build painted near-all of Colombia amber). The **2001/2002 cohorts** (Hansen's first years absorb the pre-2001 clearing backlog) are **noise pre-aged** so they don't ignite in unison: a decorrelated per-pixel age offset spreads them across their lifecycle and a tunable fraction reads as long-cooled **grey/established land** (`baseAge`/`baseGrey` knobs in the `?debug` panel) — a consequence is the national cumulative view reads greener/fainter since that backlog is a large share of total loss. A unified `defSpot` selector (`spotDim`/`spotCode` uniforms) spotlights one code from any dimension (driver/kind/legality/coca) by unpacking B with GLSL integer bit-ops. Scrubbing/spotlight only mutate std140 uniforms; `getTileData` identity is kept stable (keyed on the PMTiles handle) so tiles never refetch. **Smoothness across LOD is engineered, not assumed**: overviews are categorical-safe (R=earliest year, B=packed of that child) and **density-conserving** (G=mean) so brightness doesn't pop when a tile sharpens; the burn-front value-noise is **geo-locked** via a `tileBounds` uniform (per-tile uv → country-relative coord) so it doesn't seam across tiles or swim on zoom; `refinementStrategy:'best-available'` keeps the coarse parent visible until the finer child loads (never blank). The `smoke-deforestation.mjs` gate samples canvas pixels — a console-error-only check misses a blank-map regression. Beneath the loss raster, `ForestLayer.ts` (sibling `BitmapLayer` subclass) renders a **jungle-green forest backdrop** from `deforestation_forest.png` (R=year-2000 canopy %, A=country mask): canopy drives green depth, a 2-octave value-noise mottles foliage, alpha clips to the country. It is a **separate layer with its own single texture** — fine, since the luma second-sampler limit is per-layer, not global. Look knobs live in `forestDebug.svelte.ts` (own `?debug` panel, anchored left); treecover2000 is a year-2000 snapshot, visualization-only.
- **Panels** — `LegendDeforestation.svelte` (annual-loss meter + a **3-way lens toggle Agricultura | Legalidad | Motores**, each a ranked-by-year hoverable list driving the spotlight) and `DeforestationReadout.svelte` (per-municipio click readout incl. its ICA cattle inventory). Year scrubber + histogram in `TimeBar.svelte` (`'deforestation'` branch; state `defPos` float scrub, `defYear` derived, `defMuni`, `defSpot`).
- **Integrity** — per-municipio hectares counted from native 30 m pixels (lat-corrected), joined on DANE code; the display pyramid/PNGs are visualization-only and labeled (stats never read off any display raster). Hansen ≠ IDEAM deforestation — labeled. **Phase 2 attributions are independent, each caveated**: ag-kind = CORINE *subsequent land cover* (proxy for use, not the agent; "pasture cover ≠ ranching"; 25 ha MMU); coca = SIMCI *monitored area*, presence-correlation not coca-driven; cattle = ICA *vaccination-cycle inventory*, correlation not cause; legality = **zonal not adjudicated** (inside a park = use not permitted there, not a verdict), single-vintage boundaries labeled. IDEAM ranking stays a "estimated ordering, not official shares" disclaimer — never fabricated percentages.

## Data integrity (non-negotiable)

This project records real victims and real elections.

- Every event record must carry a source citation (dataset + ID or URL).
- Never fabricate, estimate, or interpolate coordinates, dates, victim counts, or vote counts. If a value is missing, leave it null and flag it.
- Pipeline transformations must be reproducible scripts, not manual edits to processed files.

## Data sources

Vetted source research + empirical audit live in `docs/data-sources.md` (read it before any pipeline work). Raw data (2.1 GB) is already downloaded via `pipeline/download_cnmh.py` and `pipeline/download_cede.py` (manifests with checksums in `data/raw/*/manifest.json`); re-run those scripts to re-fetch. Summary:
- Violence: CNMH SIEVCAC (primary; 11 modalities, event-level, DANE codes + complete coordinates; two cuts — 2024-09-30 CC BY-SA 4.0 licensed, 2026-03-31 current but license unstated), UCDP GED (1989–2024, names only + coords, 93% municipality-precision or better). ACLED rejected (2018+ only, restrictive EULA).
- Elections: CEDE Resultados Electorales (CC0; camara/senado/presidencia, **municipio-level, candidate-level, every election 1958–2022**, `codmpio` DANE keys + legacy crosswalk).
- Joint window at municipio level: **1958–present** (audit-verified; the earlier "~1993" research estimate was wrong). **Decision (2026-06-05): display window is 1958–present.**
- Deforestation: Hansen Global Forest Change **GFC-2025-v1.13** `lossyear` (30 m annual tree-cover loss, 2001–2025; render + per-municipio stats) + curated IDEAM national figures (official deforestation authority). Scoping research and realized-MVP notes in `docs/data-sources-deforestation.md`. Window: **2001–2025**.
- Gotchas: CNMH `Anio_hecho=0` = unknown date (65% of RU casos) — null, never impute; post-1991 Senate files are national-circunscription (0.3–2M rows, candidate × municipio).

Geographic join key: Colombian municipalities use **DIVIPOLA codes** (DANE). Prefer joining on DIVIPOLA code, never on municipality name (names are ambiguous/duplicated across departments).
